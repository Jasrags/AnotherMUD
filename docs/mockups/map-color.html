Map color & identifiers — design mockup

Two changes to the minimap and the map verb: color the terrain (and fill the missing cave / grassland / herb-garden glyphs), and overlay point-of-interest identifiers — shops, trainers, inns — derived from each room's pinned NPCs. Glyphs stay distinct so the map still reads on a no-color client; color is additive.

Out here terrain color carries it: a green forest thinning to grey ^ mountains, a % mine cutting underground, and an optional ! marker for a known hazard (the boar).

The full vocabulary it would add

@ you $ shop / vendor T trainer + inn / rest ! quest / hazard , grassland * forest ^ mountain ~ water = road o indoors % cave / underground

Identifiers are distinct glyphs, not color alone — so the map stays legible on mono / no-color clients and for color-blind players (the engine already degrades color to plain). Color is the second channel. New terrain glyphs fill today's gaps: , grassland, a cave glyph, and a herb-garden mark. POI source (recommended): derive at load from each room's pinned shop / skill_trainer NPCs — automatic, no authoring, no per-render cost.

Mockup only — colors are indicative (final values come from the theme registry, which degrades per client color tier). Layout uses the real Emond's Field / Westwood room graph.
